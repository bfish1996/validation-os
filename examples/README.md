# Examples

How the skills behave in practice, on one worked thread: a single
assumption followed from first mention to a logged decision.

**The company is fictional** — Ledgerly, two founders (Alex and Sam)
building automated bank reconciliation for sub-20-employee businesses —
inspired by the authors' real startup, with all names, numbers, and quotes
invented.

Read in order. Each file is a short scene: what you type, the moments the
skill pushes back (the `▎` blocks), what lands in the register, and the one
behaviour the scene demonstrates.

| # | Scene | Skill (entry point) |
|---|---|---|
| [01](01-setup.md) | Set up the workspace | `/setup-validation-os` |
| [02](02-seed-from-a-call.md) | Seed assumptions from a call | `/assumptions` (seed) |
| [03](03-grill-an-assumption.md) | Grill one to guardrail-complete | `/assumptions` (single) |
| [04](04-find-evidence.md) | Sweep existing evidence | `/find-evidence` (desk) |
| [05](05-design-the-experiment.md) | Design the test | `/experiment-design` |
| [06](06-meeting-prep.md) | Prep the interview | `/meeting-prep` |
| [07](07-log-the-decision.md) | Log the decision | `/decisions` (capture) |

Not shown: the maintenance entry points (assumptions audit / loop / triage,
decision sweep / audit, terminology) — each skill's SKILL.md mode table
covers them.

The thread here is product-flavoured, but the method isn't product-only —
how the same loop runs on sales outreach, pricing, fundraising, and
partnerships: [docs/domains.md](../docs/domains.md).
